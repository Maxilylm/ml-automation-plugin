# Team Review

## When to Use
- Running comprehensive code reviews with multiple specialists
- Reviewing ML pipelines, data processing, or deployment code
- Getting feedback from code quality, ML theory, and UX perspectives

## Workflow

1. Identify changed files and scope
2. Spawn parallel reviewers (code quality, ML theory, frontend UX, etc.)
3. Collect and synthesize feedback
4. Present findings with severity levels
5. Approval or revision decision
